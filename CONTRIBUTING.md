# Contributing to Quick Spring Starter 🛡️

First off, thank you for considering contributing to Quick Spring Starter! It’s people like you who make this library better for the entire Spring Boot community.

### 📜 Code of Conduct
By participating in this project, you agree to abide by our Code of Conduct. Please be kind and respectful.

### 🐛 How Can I Contribute?

#### Reporting Bugs
* Check the **Issues** tab to see if the bug has already been reported.
* If not, open a new issue. Include your Spring Boot version, Java version, and a clear description of how to reproduce the error.

#### Suggesting Features
* We love new ideas! Whether it's support for RSA, new claim mappings, or better logging—open an issue with the tag `enhancement`.

### 🛠️ Development Setup
To work on this library, you will need:
* **Java 17** or higher
* **Maven 3.8+**
* An IDE (IntelliJ IDEA is recommended)

**Steps to set up:**
1. Fork the repository on GitHub.
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/quick-spring-starter.git`
3. Run the initial build: `mvn clean install`

### 🧪 Testing Guidelines
* All new features must include unit or integration tests.
* Ensure all tests pass before submitting a Pull Request: `mvn test`

### 🚀 Pull Request Process
1. Create a new branch from `main`: `git checkout -b feat/your-feature-name`.
2. Commit your changes with descriptive messages.
3. Push to your fork and submit a **Pull Request**.
4. We will review your code as soon as possible!

### 🎨 Coding Standards
* We follow standard **Java naming conventions**.
* Please use **Lombok** for boilerplate reduction where appropriate.
* Ensure your code is formatted correctly (Standard IntelliJ format).
